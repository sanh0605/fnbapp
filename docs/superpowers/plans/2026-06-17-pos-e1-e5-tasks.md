# Technical Implementation Plan: POS E.1 Bug Fix & E.5 UI Revamp

This plan outlines the specific steps required to resolve the E.1 data loss bug in the Order Edit flow and to restructure the POS UI according to the E.5 specifications (Options A, B, C, and D).

> **Note to Implementing Agent:** Execute these tasks sequentially. Verify each step before proceeding to the next.

## Part 1: Fix E.1 Bug (Discount Data Loss in Admin Edit)

**Context:** Currently, `app/admin/orders/OrderEditModal.tsx` handles manual discounts but writes them into the `line_discount` field. This overwrites the system promo data (which is supposed to live in `line_discount`), causing data loss. We recently added `line_manual_discount` to the DB schema to separate these.

### Task 1.1: Update `EditLineItem` Interface in Backend Action
- **File:** `app/actions/order-edit.ts`
- **Action:** Update the `EditLineItem` interface (around line 6) to match the new schema separated fields.
- **Implementation:**
  ```typescript
  interface EditLineItem {
    product_id: string;
    variant_id: string;
    qty: number;
    unit_price: number;
    modifiers: any[];
    line_discount: number;         // Preserved promo portion
    line_manual_discount: number;  // Cashier/Admin entered manual portion
    discount_type: string;
  }
  ```

### Task 1.2: Update `insertMany` Payload in `editOrder`
- **File:** `app/actions/order-edit.ts`
- **Action:** Find the loop where new `Order_Lines` are prepared for `insertMany` (around line 60). Update the object pushed to `orderLinesToInsert` to use the new fields.
- **Implementation:**
  ```typescript
  // Replace:
  // line_discount: item.discount_amount || 0,
  // With:
  line_discount: item.line_discount || 0,
  line_manual_discount: item.line_manual_discount || 0,
  ```

### Task 1.3: Update `OrderEditModal.tsx` to Send Separated Fields
- **File:** `app/admin/orders/OrderEditModal.tsx`
- **Action:** Find the `handleSave` function where `editData` is constructed (around line 250). Ensure the payload matches the updated `EditLineItem` interface.
- **Implementation:**
  ```typescript
  // Inside the items.map loop in handleSave:
  return {
    product_id: item.product_id,
    variant_id: item.variant_id,
    qty: item.qty,
    unit_price: item.unit_price,
    modifiers: item.modifiers,
    line_discount: item.line_discount, // Pass through the preserved promo
    line_manual_discount: itemDiscountVND, // The newly calculated manual discount
    discount_type: "VND",
  };
  ```

## Part 2: Execute E.5 POS UI Revamp

**Context:** The `components/POSScreen.tsx` UI needs a revamp to clearly distinguish between system promotions (`PRODUCT_DISCOUNT`) and manual cashier discounts, providing better visibility and preventing overlapping errors.

### Task 2.1: Option B - Prominent Promo Banner
- **File:** `components/POSScreen.tsx`
- **Action:** Add a persistent banner at the top of the Cart Sidebar when a system promotion is active.
- **Implementation:** Locate the rendering of the Cart Sidebar (typically below the search bar or tab headers). Add a conditional block:
  ```tsx
  {appliedPromo && (
    <div className="bg-emerald-50 border-b border-emerald-100 p-3 flex flex-col">
      <div className="flex items-center gap-2 text-emerald-800 font-bold text-sm">
        <span>⚡ Áp dụng mã: {appliedPromo.name}</span>
      </div>
      <span className="text-xs text-emerald-600 mt-1">
        Giảm {appliedPromo.discount_type === 'PERCENT' ? `${appliedPromo.discount_value}%` : `${Number(appliedPromo.discount_value).toLocaleString('vi-VN')}đ`} cho các món hợp lệ
      </span>
    </div>
  )}
  ```

### Task 2.2: Option A - Cart Item Breakdown
- **File:** `components/POSScreen.tsx`
- **Action:** Update the rendering loop for `cart` items to show a breakdown of Base Price, Green System Promo, Red Manual Discount, and Orange Final Price.
- **Implementation:** Within the `cart.map` rendering block, replace the generic discount display with specific blocks. Ensure you read from `item.promo_discount` and `item.discount_amount` (manual).
  ```tsx
  {/* Price Breakdown block */}
  <div className="text-right flex flex-col items-end">
    {/* Base Price (Crossed out if any discount exists) */}
    {(item.promo_discount > 0 || item.discount_amount > 0) && (
      <span className="text-[11px] text-gray-400 line-through">
        {((item.unit_price + modsPrice) * item.qty).toLocaleString("vi-VN")}đ
      </span>
    )}
    
    {/* Green System Promo */}
    {item.promo_discount > 0 && (
      <span className="text-xs text-emerald-600 font-medium">
        KM: -{item.promo_discount.toLocaleString("vi-VN")}đ
      </span>
    )}
    
    {/* Red Manual Discount */}
    {item.discount_amount > 0 && (
      <span className="text-xs text-red-500 font-medium">
        Giảm: -{item.discount_amount.toLocaleString("vi-VN")}đ
      </span>
    )}
    
    {/* Orange Final Price */}
    <span className="font-bold text-orange-600">
      {Math.max(0, ((item.unit_price + modsPrice) * item.qty) - (item.promo_discount || 0) - (item.discount_amount || 0)).toLocaleString("vi-VN")}đ
    </span>
  </div>
  ```

### Task 2.3: Options C & D - Checkout Section Separation
- **File:** `components/POSScreen.tsx`
- **Action:** Revamp the footer/checkout summary section to clearly separate Subtotal, System Promo (Read-only), Order-level Manual Discount (Editable), and Total.
- **Implementation:** Modify the footer layout (usually the bottom sticky `div`).
  1. **Subtotal**: `calculateSubtotal()` (Sum of base prices).
  2. **System Promo (Read-only)**: Sum of all `item.promo_discount` across the cart. Display in Green.
  3. **Order-level Manual Discount (Editable)**: The existing input for `orderDiscount`. Display in Red.
  4. **Massive Orange Total**: `calculateTotalAmount()`. Make the font size `text-2xl` or `text-3xl` and color `text-orange-600`.

  ```tsx
  {/* Example Footer Structure */}
  <div className="bg-white border-t p-4 space-y-2">
    <div className="flex justify-between text-sm text-gray-600">
      <span>Tạm tính (Chưa CK)</span>
      <span>{calculateSubtotal().toLocaleString("vi-VN")}đ</span>
    </div>
    
    {/* Read-only Promo Sum */}
    {cart.reduce((sum, item) => sum + (item.promo_discount || 0), 0) > 0 && (
      <div className="flex justify-between text-sm font-medium text-emerald-600">
        <span>Khuyến mãi hệ thống</span>
        <span>-{cart.reduce((sum, item) => sum + (item.promo_discount || 0), 0).toLocaleString("vi-VN")}đ</span>
      </div>
    )}

    {/* Editable Manual Order Discount */}
    <div className="flex justify-between items-center text-sm font-medium text-red-500">
      <span>Giảm giá thêm (Thủ công)</span>
      <div className="flex items-center gap-2">
         {/* Re-use existing toggle and input for orderDiscount */}
      </div>
    </div>

    {/* Massive Orange Total */}
    <div className="flex justify-between items-center pt-2 border-t border-gray-100 mt-2">
      <span className="font-bold text-lg text-gray-800">TỔNG CỘNG</span>
      <span className="text-3xl font-black text-orange-600">
        {calculateTotalAmount().toLocaleString("vi-VN")}đ
      </span>
    </div>
  </div>
  ```
