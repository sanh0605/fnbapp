## Hypotheses to test

For each hypothesis: state whether evidence supports/rejects it, with specific data.

### H1: Historical backfill / migration artifact
- Affected orders/lines may have been written by a backfill path that used a different MAC computation than current production.
- **Test**: For each of the 170 lines, check `Order_Events.event_type IN ('MIGRATED', 'IMPORTED')` or `Orders_V2.migration_notes IS NOT NULL`. Compare stored cost_at_sale vs current replay delta for these vs non-migrated lines.
- Already partially addressed in baseline (only 2/170 lines have migrated markers). Reconfirm and dig deeper.

### H2: Backdating beyond Task 3.2 detection scope
- Task 3.2 detection matched 34/170 lines. The other 136 may be affected by backdating that didn't match the current 5-min threshold or item-match logic.
- **Test**: Query `Stock_Ledger` rows where `effective_date < created_at` (i.e., backdated) for items consumed by the 170 affected lines. Cross-reference timestamps with sale times.
- Specifically check `PO_RECEIPT`, `STOCK_ADJUST`, `PRODUCTION_YIELD` row types.

### H3: Recipe snapshot drift (line recipe != current recipe)
- Order lines store `recipe_snapshot_json`. If recipes changed after sale, current replay may use different consumption structure.
- **Test**: For each mismatched line, parse `recipe_snapshot_json`, compare ingredient list + quantities with current `Recipes` row for the same product/variant. Flag mismatches.

### H4: MAC engine code drift (commit after sale changed pricing algorithm)
- Mac-cogs computation has been refactored multiple times.
- **Test**: `git log --oneline lib/mac-cogs*.ts lib/inventory-consumption.ts lib/pos/*checkout*` — identify commits between 2026-06-20 and 2026-07-05 that could change COGS computation. Cross-reference commit dates with peak drift date (2026-06-26).

### H5: Semi-product batch yield or conversion change
- BTP_SHORTFALL pattern is dominant (132/170). Recipe or batch yield changes for semi-products would affect decomposition.
- **Test**: Compare `Semi_Products.batch_yield` history (if tracked) and `Recipes` for semi-products consumed by affected lines. Check if any semi-product recipe was edited after peak drift date.

### H6: Peak-date-specific data event
- 72% of drift concentrates on 2026-06-26. Suggests batch event (large import, recipe migration, PO bulk entry).
- **Test**: List all `Stock_Ledger` entries and `Recipes`/`Semi_Products` edits on 2026-06-26. Look for anomalies (bulk PO, mass recipe update, migration script run).
