# Task: Task 3.5 — Cohort-Aware MAC Drift Baseline Audit

## Context

`scripts/audit-mac-drift-baseline.ts` currently produces a flat list of all live MAC drift mismatches without referencing `audit_baseline_locks`. Two operational problems:

1. **Frozen artifact overwrite**: Each run overwrites `docs/audits/2026-07-09-mac-drift-baseline-lines.json` (the 170-line frozen baseline from E2). The file was reviewed and used as input to E3 + Task 3.7 + Task 3.9. Overwriting loses the reviewed provenance.

2. **No cohort classification**: As of 2026-07-16, `audit_baseline_locks` contains 436 rows across 4 cohorts:
   - 170 E3 baseline (40 PURCHASE_COST_RECOVERY + 34 BACKDATED_LEDGER + 96 UNRESOLVED)
   - 225 Task 3.7 BTP_RECIPE_REPLAY_DRIFT
   - 41 Task 3.9 BACKDATED_LEDGER_HISTORICAL_GAP

   The current script shows raw mismatch count (396+ today, grows as new drift appears) without distinguishing "locked + understood" from "new + needs investigation". Each run requires manual cross-reference.

## Goal

Refactor `scripts/audit-mac-drift-baseline.ts` + supporting library code to:

1. **Cross-reference `audit_baseline_locks`** table (read-only SELECT).
2. **Categorize each live mismatch**:
   - `LOCKED_MATCHED` — line is in `audit_baseline_locks` AND current `stored_cost_at_sale` equals `lock.stored_cost_at_sale` AND current replay equals `lock.expected_cost_at_sale`. Cohort understood, no action.
   - `LOCKED_VIOLATION` — line is in `audit_baseline_locks` BUT current stored or replay drifts from locked values. **Critical: trigger probe may have been bypassed, or stored COGS was modified.** Must never silently pass.
   - `KNOWN_NOT_LOCKED` — line matches a known cohort source_hash (Task 3.4/3.6/3.8 artifacts) but not yet locked. (Today: 0 expected. Future: pre-lock cohort.)
   - `NEW_INVESTIGATION_NEEDED` — line not in any cohort. This is the actionable population.
3. **Do NOT overwrite frozen artifacts**:
   - `docs/audits/2026-07-09-mac-drift-baseline-lines.json` — frozen, never written by this script again.
   - New output path: `docs/audits/<YYYY-MM-DD>-mac-drift-baseline-audit.json` (date-stamped, append-only by date).
4. **Output structured report**:
   - Summary counts by category
   - LOCKED_VIOLATION list (if any) — top of report, requires immediate review
   - NEW_INVESTIGATION_NEEDED list — actionable
   - Cohort breakdown (which cohort each LOCKED_MATCHED line belongs to)
5. **Stdout summary** for quick operator scan:
   - Total mismatches
   - By category counts
   - "Audit clean" if NEW_INVESTIGATION_NEEDED = 0 AND LOCKED_VIOLATION = 0
